# LESSONS

## _archive/ и Jekyll (2026-07-17)

Папка `_archive/` (старые страницы `texts/`, `styles.css`, `script.js`, `header.js`, `photo.jpg` — архив, не для публикации) не появляется на живом сайте, потому что в репозитории нет `.nojekyll`, а значит GitHub Pages собирает сайт через Jekyll со стандартными настройками. По умолчанию Jekyll исключает из сборки любые файлы и директории, начинающиеся с `_` (если явно не прописать `include:` в `_config.yml`, которого в репозитории тоже нет).

**Не добавлять `.nojekyll` в этот репозиторий.** Если файл появится, Jekyll-сборка отключится, GitHub Pages начнёт отдавать файлы как есть — и `_archive/` со всем содержимым станет публично доступна по своим путям (`/_archive/texts/...`).

Проверено на живом сайте после пуша и пересборки Pages: `/texts/music.html` → 404, `/` → 200, `robots.txt` отдаёт актуальную версию.
